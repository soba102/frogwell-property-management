Features

実装済みの機能

すべてスプレッドシート上部のメニュー「AI管理ツール」から実行できます。

1

請求書の AI-OCR 取込

Drive に置いた請求書（PDF・画像）を Gemini が読み取り、請求日・物件名・費目・金額・支払先などを自動でスプレッドシートに転記します。手入力ゼロ。

AI-OCR自動転記PDF / 画像対応
2

物件→オーナー 自動紐付け

読み取った物件名をマスターと照合し、該当オーナーを自動で紐付け。表記ゆれ（部屋番号・略称）にも対応。読取信頼度に応じて「要確認」を自動フラグ。

自動マッチング要確認フラグ
3

オーナー別レポート自動生成

ボタン1つで、オーナーごとの月次収支レポート（収入の部・支出の部・お受取額）を PDF で自動生成し、Drive に保存します。賃料収入や入金状況も反映。

PDF自動生成管理手数料計算
4

レポート一括メール送信

生成したレポートPDFを、オーナーマスターのメールアドレス宛に自動で一括送信。件名・本文も自動生成され、会社情報の署名付きで送られます。

一括送信テンプレ自動生成
5

銀行入金の自動消込

銀行CSV（SMBC等）を取り込み、振込名義からテナント→物件→オーナーを特定。賃料（請求額）と入金額を突合し、消込ステータスを自動判定します。

名寄せ自動消込滞納検知
6

ダッシュボード

請求書の処理状況（確認済 / 要確認 / AI読取済）と、入金消込の状況（消込済 / 一部入金 / 未入金 / 要確認）を件数サマリーで一覧表示します。

進捗の見える化
Reconciliation

入金消込のしくみ（注目機能）

銀行CSVの振込名義は「IBG TRANSFER」「DUITNOW」などの接頭辞やカナ表記でブレますが、 正規化した部分一致でテナントを特定し、賃料と突合して自動でステータスを付与します。

銀行CSVの振込（例）特定したオーナー賃料 / 入金判定
TRANSFER FROM AHMAD HASSAN
RM 3,500		田中太郎 様3,500 / 3,500消込済
DUITNOW LIM WEI MING
RM 2,800		鈴木花子 様2,800 / 2,800消込済
IBG TRANSFER RAJ KUMAR
RM 2,000		山田一郎 様2,500 / 2,000一部入金
（入金なし）佐藤次郎 様4,200 / 0未入金
DUITNOW XYZ TRADING SDN BHD
RM 1,500		―（特定不可）― / 1,500要確認
ポイント： 滞納（未入金）や、マスターに無い振込（要確認）が自動で炙り出されるため、 これまで目視で突合していた入金確認作業が大幅に削減されます。

Drive Folders

Google Drive フォルダの使い方

用途ごとに 3つのフォルダ を Drive に用意します。それぞれのフォルダIDを「設定」シートに登録すると、 システムが自動で読み書きします。用途別に分けるのがポイントです（混在すると誤読み込みの原因に）。

📥

① 請求書フォルダ

INVOICE_FOLDER_ID

OCRで読み取らせたい請求書を入れる場所。

▶ 入れるもの：請求書の PDF・画像（JPEG/PNG）
◀ システムの動き：新規ファイルを読取→「請求書データ」へ転記
📄

② レポート出力フォルダ

REPORT_FOLDER_ID

生成したオーナー別レポートが保存される場所。

▶ 入れるもの：なし（出力先として空で用意）
◀ システムの動き：レポートPDFを自動保存
🏦

③ 入金CSVフォルダ

DEPOSIT_FOLDER_ID

銀行からダウンロードした入金明細CSVを入れる場所。

▶ 入れるもの：銀行の入金明細 CSV
◀ システムの動き：CSVを読取→「入金消込」で突合

フォルダIDは、Drive でフォルダを開いたときのURLから確認できます。

https://drive.google.com/drive/folders/1ABCxyz123456789
　　　　　　　　　　　　　　　　　　　　　↑ この部分がフォルダID
セキュリティのヒント： 各フォルダは「自分（運用者）」と「処理を実行するアカウント」だけが アクセスできる権限にしておきます。請求書・入金明細は機密情報のため、共有範囲は最小限に。
Settings Sheet

「設定」シートの使い方

システムの動作に必要な値は、すべて「設定」シートの「値」列に入力します（左の「キー名」は変更しない）。

キー名入力する値取得・設定方法
GEMINI_API_KEYGemini の APIキーGoogle AI Studio で「Create API key」して取得
INVOICE_FOLDER_ID請求書フォルダのIDDrive のフォルダURLからコピー
REPORT_FOLDER_IDレポート出力フォルダのID同上
DEPOSIT_FOLDER_ID入金CSVフォルダのID同上
COMPANY_NAME自社名（メール署名に使用）任意で入力
COMPANY_EMAIL / COMPANY_TEL連絡先（メール署名）任意で入力
MANAGEMENT_FEE_PCT管理手数料率（％）例：8 → 賃料の8%をレポートに自動計上
キーが足りないときは「AI管理ツール」→「サンプルマスター投入」を一度実行すると、 不足している設定キーの行が自動で追加されます。
Setup

はじめの準備（セットアップ手順）

初回だけ、この順番で準備します。2回目以降は不要です。

Drive に3つのフォルダを作る

「請求書」「レポート出力」「入金CSV」用のフォルダをそれぞれ作成します。

スプレッドシートで初期セットアップを実行

「AI管理ツール」→「初期セットアップ」。必要なシートが一式作成されます（初回は権限承認のダイアログで「許可」）。

サンプルマスターを投入

「AI管理ツール」→「サンプルマスター投入」。デモ用の物件・オーナー・テナントと、不足設定キーが入ります。

「設定」シートに値を入力

上の表のとおり、APIキーと3つのフォルダID、会社情報を入力します。

マスターを自社データに差し替え（本番時）

「マスター_物件 / オーナー / テナント」を実際の情報に置き換えます。テナントの振込名義は銀行CSVに出る名前で登録するのがコツ。

Monthly Flow

毎月の運用の流れ

準備が済めば、毎月の作業は「ファイルを置く → ボタンを押す」の繰り返しです。

請求書を「請求書フォルダ」に入れる

届いた請求書のPDF・画像をアップロード。

OCR バッチ実行

「AI管理ツール」→「OCR バッチ実行」で「請求書データ」に自動転記。「要確認」行だけ目視チェック。

銀行CSVを「入金CSVフォルダ」に入れて消込

銀行から落とした入金明細CSVを入れ、「入金CSV取込（消込）」を実行。未入金・要確認を確認。

レポート生成 → メール送信

「レポート生成（全オーナー）」でPDFを作成し、「レポート一括メール送信」でオーナーへ送付。

Impact

導入前 / 導入後

これまで（手作業）

  • 請求書を1枚ずつ目視でExcelに手入力
  • オーナーごとに手でフィルタしてレポート作成
  • 銀行明細とテナントを目視で突合・消込
  • 滞納の発見が遅れる／転記ミスが起きる
  • 件数が増えるほど工数が比例して増加

このプロトタイプ（自動化）

  • ファイルを置けばAIが自動でデータ化
  • ボタン1つでオーナー別レポートを一括生成
  • 銀行CSVから自動で名寄せ・消込
  • 未入金・要確認を自動で炙り出し
  • 件数が増えても操作はほぼ同じ
Scope & Next

現時点の制約と今後の拡張余地

本プロトタイプは「動いて確認できる」ことを目的にしたもので、運用に向けては以下が拡張ポイントです。

  • 名義照合はルールベース：現状は振込名義の部分一致。カナや略称のブレはAI（Gemini）名寄せで吸収する拡張が可能。
  • 消込は当月単位：複数月にまたがる入金・前受/未収の繰越は今後の対応範囲。
  • オーナーへの送金管理（後工程）：「管理会社→オーナー」への振込・残高管理は未実装（次フェーズ候補）。
  • マスター整備が前提：物件・オーナー・テナントのマスター精度が自動化品質に直結。
  • 通貨はMYR想定：複数通貨・為替の扱いは要件に応じて拡張。
まずは実データのサンプル（請求書・銀行CSV・現状のレポート様式）をいただければ、 実運用に近い形でのチューニングへ進められます。
