# 第一弾案件 ヒアリングリスト

**不動産管理業務 AI自動化**（請求書・レポート・消込／問い合わせ対応）

---

## ご案内

請求書・テナント契約一覧・オーナーリストは**拝受済み**です。ありがとうございます。

実物を拝見して具体的な論点が見えてきましたので、**セクション 5「具体的な確認事項」**を中心にご回答いただけると助かります。残りのサンプル（レポート・入金明細）も合わせてお願いできれば、自動化の形が一気に固まります。

- 氏名・金額などは**マスキング（黒塗り）**いただいて構いません。
- **回答用スプレッドシート**: [`06_第一弾_ヒアリングリスト_回答用.csv`](./06_第一弾_ヒアリングリスト_回答用.csv) を Google スプレッドシートに取り込み、「回答（先方記入）」列にご記入ください。

| ✅ 拝受済 | 請求書のサンプル | 電気(TNB)・水道・管理費・修繕・Assessment fee 等を確認しました。追加の種類があればお願いします |
| ✅ 拝受済 | テナント契約情報一覧 | 物件名・区画番号・契約期間・家賃・敷金 等のフォーマットを確認しました |

| ⬜ 未受領 | 入金明細（消込の検証用） | 銀行の CSV・通帳の写し・入金一覧など。実物（CIMB 等）を 1 ヶ月分 |

## 5. サンプルを踏まえた具体的な確認事項 【最優先】

モック開発で判明した論点です。**ここが最も重要**です。

| # | 項目 | 質問内容 |
|---|------|----------|
| 5-1 | 請求書ファイルの命名ルール | いただいた電気代は本文に部屋番号が無く、ファイル名 `TNB Mews A-36-1_May2026` にだけ部屋番号が入っていました。**ファイル名に必ず「建物名＋部屋番号」を入れる運用ですか？** ルールが決まっていれば教えてください |
| 5-2 | 建物名の正式名称と別表記の一覧 | 請求書では建物名が住所表記でしか出ないことがあります（例：電気代は `PANGSAPURI SERVIS MEWS` ＝ The Mews）。**各建物の正式名 ⇔ 請求書/銀行で出てくる表記**の対応が分かると照合精度が上がります |
| 5-3 | 口座番号での部屋特定 | 電気・水道は請求書に契約口座番号（NO. AKAUN 等）が必ず載ります。**「口座番号 ⇔ 部屋」の対応表**があれば最も確実に紐付けできます（あれば理想、無くても可） |
| 5-4 | 部屋番号の表記ルール | オーナーリストに `29`（ハイフン無し）`20-G` `T1-19-3a` 等の揺れがありました。**正式な表記（ハイフン有無・大文字小文字）**に決まりはありますか？ |
| 5-5 | 家賃入金時の銀行摘要 | 消込はこの摘要が鍵です。**家賃が振り込まれた時、明細の摘要に「部屋番号」は必ず入りますか？**（入らない／テナント名のみ のケースがあれば教えてください） |
| 5-6 | 銀行明細の取得形式 | CIMB 等の明細は **CSV でダウンロードできますか？ それとも PDF のみ**ですか？使っている銀行名も教えてください |
| 5-7 | オーナーへのレポート送付先 | オーナーリストにメールが無かったため、**各オーナーのレポート送付先メール**をいただけますか（自動送信に使います） |
| 5-8 | 管理手数料・費用負担のルール | **管理手数料率はオーナー一律ですか／個別ですか。** 修繕費など、オーナー負担とテナント負担の切り分けルールも教えてください |
| 5-9 | 同名・法人/個人オーナーの区別 | `Ito`（個人）と `Hideo Ito Holdings Sdn Bhd`（法人）は別オーナーという理解で合っていますか？同名オーナーの区別方法があれば |

## 進め方

1. **セクション 5** の具体確認と、残りのサンプル（レポート・入金明細）をいただければ、実際に動くモックに反映してご確認いただけます。
2. セクション 2〜4 は分かる範囲で大丈夫です。
3. お渡しいただく資料に迷ったら「とりあえず現物のコピー」で問題ありません。整形・要約はこちらで行います。
